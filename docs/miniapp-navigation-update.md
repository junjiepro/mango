## 更新内容

### 在顶部导航栏添加 MiniApp 入口

为了方便用户管理和访问小应用,在应用的顶部导航栏中添加了"小应用"导航项。

## 功能特性

### 1. 导航位置 ✅

- **位置**: 顶部导航栏,在"对话"和"个人信息"之间
- **顺序**: 对话 → **小应用** → 个人信息
- **可见性**: 仅登录用户可见

### 2. 视觉设计 ✅

- **图标**: 立方体图标(与小应用主题一致)
- **文字**: "小应用"
- **激活状态**: 当前在小应用页面时,按钮显示为 `secondary` 变体(高亮)
- **响应式**: 在移动设备上隐藏(使用 `hidden md:flex`)

### 3. 交互行为 ✅

- **点击**: 导航到 `/miniapps` 页面
- **激活检测**: 使用 `isActive('/miniapps')` 检测当前路径
- **路径匹配**: 支持 `/miniapps` 及其子路径(如 `/miniapps/[id]`)

## 导航栏结构

### 完整的导航项

```
┌─────────────────────────────────────────────────────────┐
│ Mango  [对话] [小应用] [个人信息]     user@email.com [登出] │
└─────────────────────────────────────────────────────────┘
```

### 响应式行为

**桌面端 (≥768px)**:
- 显示所有导航项
- 图标 + 文字

**移动端 (<768px)**:
- 导航项隐藏
- 只显示 Logo 和用户操作区
- 用户需要通过其他方式访问(如直接输入URL或从对话中跳转)

## 用户体验改进

### 改进前 ❌

用户访问小应用的方式:
1. 在对话中让 Agent 创建小应用
2. 手动输入 URL `/miniapps`
3. 从对话中的小应用选择器跳转

**问题**:
- 不直观,用户可能不知道有小应用功能
- 访问路径不明确
- 管理小应用不方便

### 改进后 ✅

用户访问小应用的方式:
1. **直接点击顶部导航栏的"小应用"按钮** ⭐ 新增
2. 在对话中让 Agent 创建小应用
3. 从对话中的小应用选择器跳转

**优势**:
- ✅ 一键访问,方便快捷
- ✅ 功能可见性高,用户容易发现
- ✅ 与其他主要功能(对话、个人信息)并列,体现重要性
- ✅ 统一的导航体验

## 页面路由

### `/miniapps` 页面功能

用户点击导航栏的"小应用"后,会进入小应用管理页面,该页面提供:

1. **两个视图模式**:
   - **Discover**: 浏览公开的小应用
   - **My Apps**: 管理自己创建的小应用

2. **主要功能**:
   - 查看已安装的小应用
   - 浏览和搜索小应用
   - 安装/卸载小应用
   - 打开小应用详情
   - 分享小应用

3. **搜索和过滤**:
   - 按名称搜索
   - 按标签过滤
   - 按类型筛选

## 用户指南

### 如何访问小应用

1. **通过导航栏** (推荐)
   - 登录后,点击顶部导航栏的"小应用"按钮
   - 进入小应用管理页面

2. **通过对话**
   - 在对话中让 Agent 创建小应用
   - Agent 会提供小应用链接

3. **直接访问**
   - 在浏览器地址栏输入 `/miniapps`

### 小应用管理

1. **查看我的小应用**
   - 点击"My Apps"标签
   - 查看自己创建的所有小应用

2. **浏览公开小应用**
   - 点击"Discover"标签
   - 浏览其他用户分享的小应用

3. **搜索小应用**
   - 使用搜索框输入关键词
   - 按标签筛选
